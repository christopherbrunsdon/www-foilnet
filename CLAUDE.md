# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Marketing website for FoilNet (a South African hair salon), built with [Hugo](https://gohugo.io/)
and the [PaperMod](https://github.com/adityatelange/hugo-PaperMod) theme (vendored under
`themes/PaperMod`). Content is a handful of static informational pages — no blog, no posts.
Production site: https://foilnet.co.za/

## Commands

- `hugo server -D` — local dev server with drafts, live reload. The config file is `hugo.yml`
  (not the default `config.toml`); Hugo picks it up automatically.
- `hugo --minify` — production build into `public/`.
- `hugo new content/<section>/<name>.md` — scaffold a page from `archetypes/default.md`
  (creates it as `draft = true`).
- `./build.sh` — the Cloudflare Workers build entry point. It pins and installs exact tool
  versions (Hugo, Dart Sass, Go, Node) then runs `hugo build --gc --minify`. Only use this to
  reproduce the deploy environment; for local work use `hugo server`.

There are no tests, linters, or a package.json — this is a pure content/template project.

## Deployment (two paths, both live)

1. **GitHub Pages** — `.github/workflows/hugo.yml` builds and deploys on every push to `main`.
2. **Cloudflare Workers** — `wrangler.jsonc` runs `build.sh` and serves `./public` as static
   assets with `404-page` not-found handling.

Hugo version is pinned in **two** places that must stay in sync: `HUGO_VERSION` in
`.github/workflows/hugo.yml` and `HUGO_VERSION` in `build.sh`. Update both together.

## Content architecture — the non-obvious part

The site does **not** use the usual list-of-links pattern. Each nav section is a single scrollable
page assembled from multiple markdown fragments, rendered inline.

- Sections live under `content/<section>/` (e.g. `overview`, `contact-us`, `price-guide`,
  `bank-details`). Each has an `_index.md` plus several small content files.
- The custom layout `layouts/_default/list-full.html` (an override of PaperMod's `list.html`)
  renders the **full `.Content` of every child page inline** on the section page, ordered by
  each page's `weight` front-matter — instead of showing summaries with links. To add a block to
  a section, drop a new `.md` file in that section folder and set its `weight`.
- The homepage is driven by `params.profileMode` / `params.homeInfoParams` in `hugo.yml`, not by
  a content file. `params.mainSections` (`overview`, `contact-us`) controls which sections feed
  the home listing.

## Layout overrides

`layouts/` contains only files that override the PaperMod theme; everything else resolves from
`themes/PaperMod/layouts/`. Current overrides: `_default/list-full.html`, `_default/baseof.html`,
`_default/rss.xml`, and the `partials/footer.html` / `partials/extend_footer.html` /
`partials/google_analytics.html` partials. Edit theme files in `themes/PaperMod` only as a last
resort — prefer adding an override in `layouts/`.

## Search

Site search is client-side via PaperMod + Fuse.js. It requires the JSON output format, enabled by
`outputs.home` in `hugo.yml`. `content/search.md` uses `layout: search`. Fuse tuning lives under
`params.fuseOpts`.

## Not tracked in git

`public/`, `resources/_gen/`, and `.cache/` are gitignored build artifacts. They may exist locally
but are regenerated by every build — do not commit them or hand-edit files inside them.
